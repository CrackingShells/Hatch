# Documentation Instructions for `docs/**/*.md`

This article describes the style guide and structure for all documentation in the `docs/` directory. It covers how to organize articles, tutorials, appendices, and resources.

You will learn about:

- The preferred style for the documentation
- Where to place different types of documentation files

## Style Guide

- Use Markdown format for all documentation files.
- Keep documentation concise and focused.
- Start each article with:
  - **This article is about**: List main concepts.
  - **You will learn about**: List outcomes or skills.
- Avoid repeating information from code docstrings; link to code or other articles instead.
- Use plain, neutral language in present tense and active voice.
- Use short sentences and bullet points.

## Structure

- Place user guides and tutorials in `docs/articles/users/` and `docs/articles/users/tutorials/`.
- Place developer and contributor documentation in `docs/articles/devs/`.
- Place appendices, glossary, and beginner resources in `docs/articles/appendices/`.
- Store images and diagrams in `docs/resources/`.
- Use PlantUML for diagrams in `docs/resources/diagrams/`.

Refer to this article when creating or editing documentation files.
