# Developer Guide

## Contribution Guidelines

This article is about:
- How to contribute to Hatch
- Contribution process and requirements

You will learn about:
- Steps for submitting issues and pull requests
- Code, test, and documentation standards

- Open issues for bugs, features, or questions (see GitHub Issues)
- Discuss new features before starting major work
- Fork, branch, and submit pull requests for code changes
- Follow code guidelines (PEP 8, type hints, docstrings)
- Write and run tests for new or changed code
- Update documentation as needed
- Ensure all CI tests pass before submitting
- See [Contributing Guide](../../devs/contributing.md) for details and examples
